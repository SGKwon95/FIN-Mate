## 분석

### 문제 1 — LLM 언어 오염 (치명적)

재작성 쿼리 10개 중 3개에서 모델이 **중국어·일본어를 섞어 출력**했다.

| 원문 | 오염된 재작성 |
|------|--------------|
| "예금자보호" | "예금자 보호制度について説明してください…" (일본어 포함) |
| "계좌 만들려면 뭐 가져가야 해" | "예금 계좌开设所需的相关金融文件和信息有哪些？" (중국어 전환) |
| "미성년자 계좌 개설 시 필요 서류는?" | "미성년자가 계좌를开设时所需文件的规定是什么？" (중국어 혼입) |

이 경우 임베딩 벡터가 한국어 문서 공간에서 완전히 이탈해 유사도가 급락한다 (0.816 → 0.688).

### 문제 2 — nomic-embed-text가 한국어 키워드를 이미 잘 처리

단어형 원문의 최고 유사도가 충분히 높다.

| 쿼리 | 원문 최고 유사도 | 원문 CR |
|------|----------------|---------|
| 보이스피싱 | 0.729 | 3/5 |
| 중도해지 | 0.750 | 4/5 |
| 예금자보호 | 0.822 | 2/5 |

`nomic-embed-text-v1.5`는 한국어 단어의 의미를 직접 이해하므로, 키워드를 장문으로 확장해도 벡터 방향에 큰 이점이 없다. 오히려 다른 키워드가 섞여 노이즈가 된다.

### 문제 3 — 재작성이 의미를 바꿀 때 CR 역전

"보이스피싱 피해구제 신청 절차는?" (CR 4) → "보이스피싱 피해 구제 신청 절차에 대해 설명하시오." (CR 2)  
두 쿼리는 거의 동일하지만 LLM 판정이 달라졌다. 이는 judge 모델(동일 모델)의 **판정 불안정성**도 있지만, 재작성 후 검색된 청크 세트가 바뀐 것이 주원인이다.
